## Куда разместить решение?

В ваш личный репозиторий, созданный через [Систему](https://contest.gitlab.prodcontest.com).

> [!WARNING]
> Помните! Все изменения должны быть строго в директории `solution` и в ветке `main`!\
> Любые другие файлы за пределами директории `solution` защищены от изменений.\
> Также защищен от изменений файл `solution/data.json`.

## Как запустить проект?

1. [Склонируйте репозиторий](https://docs.github.com/ru/repositories/creating-and-managing-repositories/cloning-a-repository).
2. Установите на устройство [Node.js](https://nodejs.org/en/download). Рекомендуется версия не ниже 24.12. Если установка прошла успешно, то эти команды в терминале выведут цифры установленных версий:
```
node -v
npm -v
```
3. Перед первым запуском откройте терминал, в нем перейдите в папку репозитория и выполните команду `npm ci`. Если установка прошла успешно, то в папке появится директория `node_modules`.
4. Запустите проект командой `npm run start`. Откройте в браузере http://localhost:8080/ - если проект запустился, то вы увидите вкладку с названием "PROD 2". Это ваше решение.

В следующие разы достаточно команды `npm run start`.

## Как запустить открытые тесты?

Есть 2 варианта: 
- Запуск с интерактивным интерфейсом, в котором можно наглядно изучить прохождение тестов.
- Запуск через Docker, в котором используется окружение как на удаленных проверках.

### Запуск с интерактивным интерфейсом

Предварительно выполните шаги 1-3 из [Как запустить проект?](#как-запустить-проект)

1. Перед первым запуском установите зависимости командой `npx playwright install`
2. Запустите [интерфейс](https://playwright.dev/docs/test-ui-mode) командой `npm run test:ui` и используйте его.

### Запуск через Docker

1. Установите [Docker](https://docs.docker.com/get-started/introduction/get-docker-desktop/)
2. Для запуска тестов выполните следующие команды в терминале:
```
docker build -t prod-task-2-tests .
docker run -it --rm -v "$(pwd)/test-results:/test-results" prod-task-2-tests
```
3. Если в процессе запуска не прошли какие-то проверки, то появится папка test-results с деталями ошибок.

> [!WARNING]
> Скриншотные тесты следует проверять запуском через Docker!
